# Claude Code for Research

A hands-on workshop for researchers — MSc, PhD, RAs — on using Claude Code
effectively for research codebases, algorithm reimplementation, and reproducible
experiments.

## What you'll learn

By the end of the course you'll be able to set up Claude Code for research use,
make a repo Claude-friendly through `CLAUDE.md`, drive Claude through TDD loops
on real research code, build one useful slash command, and audit AI-generated
output before it lands in a paper or experiment.

The single takeaway, if you forget everything else: **make the repo
Claude-friendly first, then drive it with constrained loops, and never accept
output you haven't read.**

## Format

Seven modules, hands-on. Total ~6–8 hours. You'll need Claude Code installed and
a laptop open for the live sessions.

## Modules

- **[Module 0 — Why this course exists](module-00-framing.md).** What Claude Code is and isn't. Realistic expectations.
- **[Module 1 — Setup & mental model](module-01-setup.md).** Install, configure defaults, multi-account, the agentic loop.
- **[Module 2 — Making a repo Claude-friendly](module-02-claude-md.md).** The single biggest lever: `CLAUDE.md`.
- **[Module 3 — Daily research workflows](module-03-workflows.md).** TDD loops, reading unfamiliar repos, paper → code, SLURM.
- **[Module 4 — Slash commands and skills](module-04-slash-commands.md).** When and how to extract automations.
- **[Module 5 — Avoiding failure modes](module-05-failure-modes.md).** Plausible-but-wrong, review patterns, reproducibility.
- **[Module 6 — Case studies](module-06-case-studies.md).** Real research repos, plus bring-your-own.

## Audience adjacency

The exercises lean PDDL-flavored because the course author works in that area
and can answer questions live. The lessons are domain-independent.

If you're working in RL, DRL, MARL, or anything else: the techniques transfer.
Module 6 includes a bring-your-own-repo option so you can apply the full pipeline
to your actual research code.

## Before you start

- Install Claude Code (covered in Module 1).
- Bring a laptop with terminal access.
- Optional: have one of your own research repos handy for Module 6.
